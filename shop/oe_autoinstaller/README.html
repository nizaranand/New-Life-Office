Autoinstaller 2.14 for:

Order Editor 5.0.6.6 ReyBlack (1 Dec 2008)



The autoinstaller script requires PHP 5.2.0

It was tested on hostgator.com host (PHP:5.2.4 MySQL 5.0.67) and
osCommerce Online Merchant v2.2 RC2a

It doesn't work for
osCommerce Online Merchant v3.0 Alpha 5


################### WHAT IT DOES #####################

The script automatically install the contribution.
The aim of the script is to speed up the contribution installation.
And to make possible to uninstall it.

You will need to do manually only what the script cannot.

################### INSTALLATION #####################

1) Make directory catalog/admin/order_editor

2) Upload the oe_autoinstaller folder to your store root :

www.yoursite.com/catalog/oe_autoinstaller

or to the site root if your store is installed to the site root:

www.yoursite.com/oe_autoinstaller


3) Point your web browser to it and click the "Install" button


################### HOW AUTOINSTALLER WORKS #####################

The script does all it can: copy files, edit files, alter database tables.

It saves a backup before any action to the oe_autoinstaller/backup directory.

If the script cannot do something it will suggest you to do it manually. Then you can click the "Skip" button to continue the installation.

If something goes wrong you will be able to click the "Uninstall" button. The script will restore all the backups and "un alter" the database.

The script cannot overwrite the backup files. So if you want to reinstall (install-uninstall-install) you will have to delete the backup files from the oe_autoinstaller/backup directory. I suggest you to save a copy of each on your computer before deleting.

################### UNINSTALLATION #####################

Point your web browser to the oe_autoinstaller directory and click the "Uninstall" button

NOTE: The uninstallation feature have sense just after the installation.
If you do any changes in the site administration area or install
another contributions and then will try to uninstall - all your changes can be
lost because of restoring the old files.
